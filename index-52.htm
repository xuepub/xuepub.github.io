Bootstrap 基础

on 2016年4月22日
  • 前端
by szzz

推荐阅读

Bootstrap 官方中文文档 http://v3.bootcss.com/

W3school 相关文档 http://w3schools.bootcss.com/bootstrap/

《Bootstrap实战》 中文版

 

有效文件

bootstrap/
├── css/
│ └── bootstrap.min.css
├── js/
│ └─── bootstrap.min.js
├── fonts/

 

引入

在 <head> 中导入 bootstrap.min.css，注意在自定义 CSS 文件之前；

在 <body> 结束部分引入  bootstrap.min.js，注意在引入 jquery.min.js 文件之后；

可以考虑使用 CDN 公共库。

 

提供功能

  • 各浏览器 reset
  • 移动优先 responsive design
  • 栅格系统
  • 导航栏
  • 表格和表单样式
  • 模态框
  • 标签页
  • 图片轮播
  • 响应式图片

 

相关组件

  • jQuery（必须使用）
  • jQuery UI
  • Font Awesome 图标

 

使用要点

移动优先

必须使用 Bootstrap 要求的三行 <meta> 以支持各种设备。

 

 js 文件后置引入

据称在 <body>  的结束部分引入 js 文件能够优化网页加载速度，提升动画效果，业界普遍使用。

注意这种情况下 DOM 事件的写法，比如无法直接在 HTML 的 button 标签中使用 onclick 事件。

 

顶部导航 nav

格式需按 Bootstrap 文档来。

移动端导航通过下拉式菜单弹出，其位置在 div.navbar-header>button.navbar-toggle.collapsed，注意 button 中 data-target 属性值为 #main-nav，main-nav 为正常界面下主菜单的 id 值，要保证 Bootstrap 的 js 文件和其依赖的 jQuery 文件正常引用。

二级菜单的写法类似移动端下拉式菜单。

 

响应式  responsive

1. container

所有网站上的内容，都放到 <div class=”container”></div> 中，这样页面宽度自动支持响应式设计：

  • lg 大屏幕 x >1200px，1170px container 宽度
  • md 中屏幕 992px < x < 1200px，970px container 宽度
  • sm 小屏幕 768px < x < 992px，750px container 宽度
  • xs 手机屏幕  x < 768px ，auto container 宽度

2. 显示/隐藏元素

在class中加入 hidden-xs、hidden-sm、hidden-md、hidden-lg，可分别在不同尺寸页面下隐藏不同元素，其相反作用使用 visible 前缀

3. row

当 div 中有多列子块时，使用栅格系统，当栅格列之和 >=12 时，移动浏览器可能产生右侧页面移动的 bug，请小心使用：

<div class="row">
  <div class="col-md-6 col-xs-12"></div>
  <div class="col-md-6 col-xs-12"></div>
</div>

Python 爬虫简述

on 2016年4月15日
  • 后端
  • 开发
by szzz

环境

OS : Windows 10 1511

Python -v : 3.5.1 win32

IDE : Pychram 2016.1.2

Test Browser : Firefox

DB : MySQL

 

模块准备

urllib 内置，获取 HTML，容易被封

Selenium 需安装，配合浏览器获取 HTML

BeatifulSoup4 需安装，文档，分析 HTML

pymysql 需安装，操作 MySQL

 

思路

优先使用 urllib 进行网页抓取，如果完全被禁，使用 Selenium，后者默认使用带 UI 的浏览器 Firefox ，可使用无 UI 浏览器取代。

Selenium / urllib 取出数据所在的 HTML 代码，交给 BeatifulSoup4 做具体抽取

可通过列表（数组）、字典（关联数组），或类来存储页面上的结构化数据

将内存中的数据通过 pymysql 存入 MySQL 数据库

 

一些细节

Selenium

设置加载页面的时间，防止被墙的脚本拖延页面加载时间  driver.set_page_load_timeout(num)

使用 XPath 获取元素 driver.find_element_by_xpath(‘//body’)

得到元素 HTML 代码 body.get_attribute(‘innerHTML’)

BeatifulSoup4

使用 urllib.parse.unquote() 对链接中的 URL 转码进行还原

SQL

使用 INSERT IGNORE INTO 取代 INSERT INTO，避免主键重复造成的错误

启用 .htaccess 实现 WordPress 固定链接

on 2016年4月3日
  • 后端
  • 运维
by szzz

在 Ubuntu Apache2 的 WordPress 网站中，固定链接需要如下设置：

  1. 启动 mod_rewrite
    • sudo a2enmod rewrite
    • sudo service apache2 restart
  2. 启用 .htaccess
    • 修改 apache2 配置文件，sudo nano /etc/apache2/sites-enabled/000-default.conf
      其中内容为
      <Directory /var/www/html>
      Options Indexes FollowSymLinks MultiViews
      AllowOverride All
      Order allow,deny
      allow from all
      </Directory>
    • sudo service apache2 restart
  3. 按 WordPress 设置 – 固定链接 – 自定义链接给出的提示，添加 .htaccess 至相应根目录

Ubuntu LAMP 部署

on 2016年3月31日
  • 后端
  • 运维
by szzz

系统

  • 阿里云，购买并开通 Ubuntu 服务器
  • Digital Ocean， 开通 Ubuntu 服务器
  • 腾讯云，购买并开通 Ubuntu 服务器
    • 由于不提供 root 密码，需要进入 ubuntu 账号后，使用sudo passwd root 更改密码
    • sudo vi /etc/ssh/sshd_config，修改 PermitRootLogin 为 yes，保存退出
    • 重启 ssh，sudo service ssh restart
  • 虚拟机 vmware，中安装 ubuntu desktop / server

 

下载工具

  • XShell 用于远程访问服务器的终端
  • WinSCP 用户远程访问服务器的文件系统

 

终端基本命令

  • cd 目录操作
  • ls 列出当前文件
  • service –status-all 查看系统当前进程

 

安装LAMP

  1. sudo apt-get update
    sudo apt-get install apache2
  2. sudo apt-get install mysql-server php5-mysql
    sudo mysql_install_db
    sudo mysql_secure_installation
  3. sudo apt-get install php5 libapache2-mod-php5 php5-mcrypt

 

启动 apache mysql

  • service apache2 start/restart/stop
  • service mysqld start/restart/stop

 

测试 php 环境

  1. 创建一个 .php 文件，内含代码 <?php phpinfo(); ?>
  2. 将该文件放入 apache 的文档根目录 /var/www/html/ 中

 

安装 phpmyadmin

  1. sudo apt-get install wget unzip
  2. 在 html 根目录下执行 wget <phpmyadmin 下载地址>
  3. unzip <phpmyadmin  下载包>

 

安装wordpress

  1. wget <wordpress 下载地址>
  2. unzip <wordpress  下载包>
  3. 安装

 

vi 基本操作

  • ESC 进入命令行模式（退出编辑）
  • i 进入插入模式（可编辑）
  • :wq 保存文件并退出
  • :q 直接退出文件

 

 

地址栏域名解析

on 2016年3月29日
  • 后端
  • 运维
by szzz

127.0.0.1 或 localhost：访问的是本机的 Web Server

192.168.x.x：局域网 Web Server

104.131.152.56：ip访问互联网

foo.com：域名访问互联网

 

域名解析至 ip 地址

  1. 在域名管理商处设置 A 记录，指向服务器根目录
  2. 设置 apache 的 vhost ，指向服务器子目录
