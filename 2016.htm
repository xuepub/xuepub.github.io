Docker 基础

on 2016/06/20
  • 后端
  • 运维
by szzz

 

Docker 是开源容器，用于更快的交付应用，典型场景有：

  • 代码构建
  • 应用部署
  • 容器即服务

 

参考

  • 官方网站 https://www.docker.com/
  • 中文 《第一本 Docker 书》
  • 中文 《Docker 即学即用》，Oreilly
  • 《Using Docker》、《Docker Cookbook》，Oreilly

 

入门

使用

  • 使用 Docker Toolbox，安装 Docker 软件
    • https://www.docker.com/toolbox
  • 使用 Docker Engine，在容器中运行软件镜像
    • $ docker images，查看本地镜像
    • $ docker run hello-world
  • 如果本地不存在镜像，则获取 Docker Hub 镜像

 

创建

  • 创建 Docker Hub 账户和镜像仓库
    • $ docker login
  • 创建自己的镜像
    • Dockerfile
    • $ docker build -t docker-name
  • push（推）镜像到 Docker Hub，让他人使用
    • $ docker push docker-name，上传
    • $ docker pull docker-name，下载

 

网络

  • 国内源 http://get.daocloud.io/
  • $ docker-machine restart default # Restart the environment
    $ eval $(docker-machine env default) # Refresh your environment settings

 

Docker Engine

容器操作

  • $ docker ps，查看所有正在运行中的容器列表
  • $ docker ps -a，查看所有未运行的容器列表
  • $ docker rm $(docker ps -a -q)，删除所有未运行的容器
  • $ docker start -i container-id
  • $ docker stop container-id

 

vm 操作

  • $ docker-machine ls
  • $ docker-machine create –driver virtualbox my-default
  • $ docker-machine rm my-default

 

使用预制 Ubuntu 镜像

  • $ docker pull ubuntu
  • $ docker run -i -t ubuntu /bin/bash
  • ctrl-p ctrl-q，detach 容器
  • ctrl+d，exit 容器
  • $ docker attach ubuntu-container-id
  • $ docker rmi -f ubuntu，删除镜像

 

制作 Java Spring Boot 镜像

  • https://spring.io/guides/gs/spring-boot-docker/
  • 中文 http://docs.daocloud.io/java-docker/docker-java-spring-boot

 

构建并使用 MongoDB 镜像

  • https://docs.docker.com/engine/examples/mongodb/

 

Docker Compose

用于运行多容器的 Docker 应用

  • $ docker-compose  up

示例，https://docs.docker.com/compose/wordpress/

on 2016/06/15

AngularJS 1 即将退役，由基于 TypeScript 的 Angular 2 取代

旨在开发 SPA 单页应用

  • 官方文档 https://docs.angularjs.org/guide
  • 入门 http://w3schools.bootcss.com/angular/default.html
  • 中文参考 https://github.com/jmcunningham/AngularJS-Learning/blob/master/ZH-CN.md

 

图书

  • 《AngularJS 学习手册》，Oreilly
  • 《用 AngularJS 开发下一代 Web 应用》，Oreilly
  • 《AngularJS 权威教程》，ng-book

 

核心概念

  • Template 模板：附加标记的 HTML
  • Directives 指令：自定义属性和元素的扩展 HTML
  • Model 模型：视图中显示给用户的数据，并与用户交互
  • Scope 域：存储模型的上下文，以便于控制器指令和表达式访问 $scope
  • Expressions 表达式：从域中访问变量和函数 {{ expression }}
  • Compiler 编译器：解析模板、初始化指令和表达式
  • Filter 过滤器：为用户显示格式化表达式值 {{ expression | filter }}
  • View 视图：用户所见（DOM）
  • Data Binding 数据绑定：在模型和视图之间同步数据
  • Controller 控制器：视图后的商业逻辑
  • Dependency Injection 依赖注入：创建并连接对象和函数
  • Injector 注入器：依赖注入容器
  • Module 模块：一个容器用于应用的不同部分，包括控制器、服务、过滤器、指令，由注入器配置
  • Service 服务：依赖视图的可重用商业逻辑

 

常用指令

  • ng-app 声明 AngularJS 应用的根元素，其中指令可用于声明绑定和定义行为
  • ng-bind 设置 DOM 元素的文本为表达式值。如 <span ng-bind=”name”></span> 在 span 元素中显示了 name 的值，任何在应用域中变量 name 的更改都立刻反应到 DOM
  • ng-model 与 ng-bind 相似，但在视图和域之间建立了一个双向数据绑定
  • ng-model-options 提供了如何做模型更新的调整
  • ng-class 让 class 属性动态加载
  • ng-controller 指定一个 JavaScript 控制器类，对 HTML 表达式求值
  • ng-repeat 从一个集合中每项实例化一个元素
  • ng-show & ng-hide 有条件的显示或隐藏元素，根据布尔表达式的值。显示和隐藏通过设置 CSS display 样式来完成
  • ng-switch 有条件的从一系列选择中实例化一个模板，根据选择表达式的值
  • ng-view 这一基本指令负责处理路由，由特定的控制器驱动，在渲染模板之前解析 JSON
  • ng-if 基本的 if 表达式指令，允许显示是否条件为 true 的元素。当条件为false，元素将从 DOM 移除。当 true 重新插入编译过的元素备份
  • ng-aria 一个模块用来支持通用 ARIA 属性
  • ng-animate 一个模块提供了 JavaScript、CSS3 过渡和 CSS3 关键帧动画的支持，连接已有的核心和自定义指令

JSX 本质上是 JavaScript 的语法转换，可创建语义丰富的标签

一些关注点：

  • JSX 用小写字母开头的标签名渲染 HTML 标签（字符串），大写字母开头的标签名渲染 React 组件（类）
  • { } 中使用 condition ? true-branch : false-branch 三目运算符 ，实现 if-else 结构
  • { } 中使用 && 运算符，当前操作数为 true 输出后操作数
  • 事件 onChange、onClick
  • 特殊属性
    • key，用于调整组件位置
    • ref，引用子组件
    • dangerouslySetInnerHTML
    • htmlFor
    • className

 

组件生命周期

  • 生命周期：实例化、活动期、销毁
  • 实例化 / 加载：getDefaultProps、getInitialState、componentWillMount、render、componentMidMount
  • 活动期 / 更新：componentWillReceiveProps、shouldComponentUpdate、componentWillMount、render、componentMidMount
  • 销毁 / 移除：componentWillUnmount

props 在 getDefaultProps 初始化时传入数据，此后只读，即单向数据流

state 中存储了视图状态，在组件事件中修改 this.setState() 是改变视图的关键点

 

复杂组件

  • 复杂组件可由多个简单组件组合而成，这些组件形成了树结构
  • 大型项目一般从下而上（也可以理解为从内到外、从小到大）实现，便于写测试
  • 组件的初始数据通过 props 传入，从上到下单向数据流动，
  • state 则存储了运行组件时变化的状态值，包括可能被组件的事件处理器改变并触发用户界面更新的数据，如搜索框内输入的文本、当前多选框的选中值
  • 让尽可能多的组件无状态化

<?php
 
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Database\Migrations\Migration;
 
class CreateBooksTable extends Migration
{
    public function up()
    {
        Schema::create('books', function (Blueprint $table) {
            $table->char('isbn', 13);          //isbn CHAR(13),
            $table->string('author', 128);     //author VARCHAR(128),
            $table->string('title', 128);      //title VARCHAR(128),
            $table->string('category', 128);   //category VARCHAR(32),
            $table->smallInteger('year');      //year SMALLINT
            $table->primary('isbn');           //设置主键
            $table->timestamps();              //时间戳
        });
    }
 
    public function down()
    {
        Schema::drop('books');
    }
}

表单添加图书流程：

  • 添加新路由 Route::post(‘books’, ‘BooksController@create’);
  • 控制器类 BooksController中实现 create() 方法
    • 方法至少传入一个参数 Request $request
    • new 实例化一个待存储对象
    • 用 $request 获取表单提交的数据，存入对象
    • 使用 Eloquent 中的 save() 方法存储数据
    • 返回值使用 redirect() 跳转到某个页面，或使用 back() 跳转到之前的页面
  • 模型类中要对存储的 isbn、author 等字段设置 protected $fillable = [‘isbn’, ‘author’]; 否则安全原因无法正常存储
  • 视图中实现添加表单 <form action=”books” method=”POST”>

 

表单修改数据流程：

  • 添加新路由 Route::get(‘books/{book}/edit’, ‘BooksController@edit’); 用于进入编辑页面
  • 添加新路由 Route::patch(‘books/{book}’, ‘BooksController@update’); 用于提交编辑结果
  • 实现编辑页面的控制器 update() 方法
    • use  App\Models\Book;
    • 传入至少两个参数  Request $request, Book $book
    • $book -> update($request -> all());
    • return back();
  • 实现视图
    • 表单 <form method=”POST” action=”books/{ $book->isbn }”>
    • 表单中加入 {{ method_field(‘PATCH’) }}

HTTP REST 简述

on 2016/05/02
  • 后端
  • 开发
by szzz

HTTP 协议

目前为 HTTP 1.1

用户要访问某个资源，通过浏览器发送请求 Request，指向了 URL / URI 对应的服务器

服务器接收到请求后，返回给用户一个响应 Response

如果访问成功，通常响应会包含页面内容，如果不成功，可能会看到 404 页面

 

HTTP 协议是无状态的

 

请求类型：OPTIONS、GET、HEAD、POST、PUT、DELETE、TRACE、CONNECT

响应状态码：1xx（消息）、2xx（成功）、3xx（重定向）、4xx（客户端错误）、5xx（服务器错误）

 

REST

REST 表述性状态转移，是当前最流行的互联网架构之一，目的是设计“完美 URL”，降低客户端和服务端的复杂度

通过设计 URL 来统一资源的表述，规定请求类型：

  • POST /uri 增加资源
  • GET /uri/{id} 查找资源
  • PUT /uri/{id} 更新资源
  • DELETE /uri/{id}  删除资源
  • （可选）PATCH

资源既可以是 HTML（提供内容），也可以是 JSON 数据（提供 APIs、Web Services）

可构造不同层次的 Web 系统

Bootstrap 基础

on 2016/04/22
  • 前端
by szzz

推荐阅读

Bootstrap 官方中文文档 http://v3.bootcss.com/

W3school 相关文档 http://w3schools.bootcss.com/bootstrap/

《Bootstrap实战》 中文版

 

有效文件

bootstrap/
├── css/
│ └── bootstrap.min.css
├── js/
│ └─── bootstrap.min.js
├── fonts/

 

引入

在 <head> 中导入 bootstrap.min.css，注意在自定义 CSS 文件之前；

在 <body> 结束部分引入  bootstrap.min.js，注意在引入 jquery.min.js 文件之后；

可以考虑使用 CDN 公共库。

 

提供功能

  • 各浏览器 reset
  • 移动优先 responsive design
  • 栅格系统
  • 导航栏
  • 表格和表单样式
  • 模态框
  • 标签页
  • 图片轮播
  • 响应式图片

 

相关组件

  • jQuery（必须使用）
  • jQuery UI
  • Font Awesome 图标

 

使用要点

移动优先

必须使用 Bootstrap 要求的三行 <meta> 以支持各种设备。

 

 js 文件后置引入

据称在 <body>  的结束部分引入 js 文件能够优化网页加载速度，提升动画效果，业界普遍使用。

注意这种情况下 DOM 事件的写法，比如无法直接在 HTML 的 button 标签中使用 onclick 事件。

 

顶部导航 nav

格式需按 Bootstrap 文档来。

移动端导航通过下拉式菜单弹出，其位置在 div.navbar-header>button.navbar-toggle.collapsed，注意 button 中 data-target 属性值为 #main-nav，main-nav 为正常界面下主菜单的 id 值，要保证 Bootstrap 的 js 文件和其依赖的 jQuery 文件正常引用。

二级菜单的写法类似移动端下拉式菜单。

 

响应式  responsive

1. container

所有网站上的内容，都放到 <div class=”container”></div> 中，这样页面宽度自动支持响应式设计：

  • lg 大屏幕 x >1200px，1170px container 宽度
  • md 中屏幕 992px < x < 1200px，970px container 宽度
  • sm 小屏幕 768px < x < 992px，750px container 宽度
  • xs 手机屏幕  x < 768px ，auto container 宽度

2. 显示/隐藏元素

在class中加入 hidden-xs、hidden-sm、hidden-md、hidden-lg，可分别在不同尺寸页面下隐藏不同元素，其相反作用使用 visible 前缀

3. row

当 div 中有多列子块时，使用栅格系统，当栅格列之和 >=12 时，移动浏览器可能产生右侧页面移动的 bug，请小心使用：

XHTML
1
2
3
4
<div class="row">
  <div class="col-md-6 col-xs-12"></div>
  <div class="col-md-6 col-xs-12"></div>
</div>

Python 爬虫简述

on 2016/04/15
  • 后端
  • 开发
by szzz

环境

OS : Windows 10 1511

Python -v : 3.5.1 win32

IDE : Pychram 2016.1.2

Test Browser : Firefox

DB : MySQL

 

模块准备

urllib 内置，获取 HTML，容易被封

Selenium 需安装，配合浏览器获取 HTML

BeatifulSoup4 需安装，文档，分析 HTML

pymysql 需安装，操作 MySQL

 

思路

优先使用 urllib 进行网页抓取，如果完全被禁，使用 Selenium，后者默认使用带 UI 的浏览器 Firefox ，可使用无 UI 浏览器取代。

Selenium / urllib 取出数据所在的 HTML 代码，交给 BeatifulSoup4 做具体抽取

可通过列表（数组）、字典（关联数组），或类来存储页面上的结构化数据

将内存中的数据通过 pymysql 存入 MySQL 数据库

 

一些细节

Selenium

设置加载页面的时间，防止被墙的脚本拖延页面加载时间  driver.set_page_load_timeout(num)

使用 XPath 获取元素 driver.find_element_by_xpath(‘//body’)

得到元素 HTML 代码 body.get_attribute(‘innerHTML’)

BeatifulSoup4

使用 urllib.parse.unquote() 对链接中的 URL 转码进行还原

SQL

使用 INSERT IGNORE INTO 取代 INSERT INTO，避免主键重复造成的错误
