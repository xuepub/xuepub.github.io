基本代码分析

com
  +- example
    +- myproject
      +- Application.java - 1
      |
      +- domain
      | +- Customer.java - 2
      | +- CustomerRepository.java - 3
      |
      +- service
      | +- CustomerService.java - 4
      |
      +- web
        +- CustomerController.java - 5

 

Application.java – 1 

入口 Main 方法，执行配置及服务

类注解 @SpringBootApplication 等同于 @Configuration 配置类 + @EnableAutoConfiguration + @ComponentScan 自动扫描

@Configuration 使用基于 XML 的配置

@ComponentScan 用于自动查找所有以 @Component、@Service、@Repository、@Controller 注册的 Bean

方法注解 @Autowired 指定构造方法注入的依赖

 

domain/Customer.java – 2

POJO，即领域模型

类注解 @Entity

存储数据对象，定义属性、 get / set 方法，可通过构造方法 set 属性

 

domain/CustomerRepository.java – 3

Spring Data JPA Repositories 数据访问

继承自 Repository<Customer, Long> 或 CrudRepository<Customer, Long> 接口

 

service/CustomerService.java – 4

服务类 / 业务逻辑类 – 封装功能

类注解 @Service

 

web/CustomerController.java – 5

Spring Web MVC 控制器

类注解 @Controller 或 @RestController

类 / 方法注解 @RequestMapping 指定路由

返回视图（JSP、模板）或 Json（前后分离）

 

配置

CustomerConfig.java

支持基于 Java 和 XML 的配置

类注解 @Configuration 声明配置类，用于定义 Bean

类注解 @ComponentScan 自动扫描

方法注解 @Bean 声明配置类中方法的返回值为 Bean

 

application.properties

功能参数配置

 

数据

http://blog.netgloo.com/2014/10/27/using-mysql-in-spring-boot-via-spring-data-jpa-and-hibernate/

参数自动配置 / Java 手动配置 DataSource

声明实体类，@Entity

声明对应的 Dao 接口，继承 CrudRepository 接口

在控制器中指定 Dao，并操纵数据对象

使用 IntelliJ IDEA 开发 Spring Web

on 2016年6月2日
  • 应用
by szzz

 准备

  • JDK 8
    • 系统配置 – JAVA_HOME、Path、CLASSPATH
  • Tomcat 8.0
  • IntelliJ IDEA Ultimate 2016.1
    • Maven 内置
  • Gradle
    • 系统配置- GRADLE_HOME

 

IntelliJ 常规配置

  • 代码字体和配色 File – Setting – Editor – Colors & Fonts
  • 服务器设置 File – Setting – Build, Excution, Deployment – Application Servers
    • 添加 Tomcat
  • File – Setting – Version Control – Github

 

创建工程

  • Create New Project – Spring Initializr
    • Maven war 1.8 Java
    • Group = package: com.xuepub
    • Name = Artifact
    • 依赖 (Dependencies): Web Thymeleaf
  • 创建文件后
    • pom.xml – Add as Maven Project
    • Maven 开始工作

 

配置工程

  • File – Project Structure – Artifacts
    • 修改 [name]:war
    • 修改 [name]:war exploded
  • Run – Edit Configurations
    • 添加 Tomcat Server – Local
      • 命名为 tomcat
      • Server – Before launch: Make, Activate tool window 加入 Build Artifacts 2 项
      • Deployment – 添加 [name]:war  或  [name]:war exploded 2 选 1

 

运行

  • Run – Run… – Tomcat – 能成功运行
    • 此时启动本机 Java Web 容器 Tomcat（采用 War 发布）
    • War 包可部署至产品服务器
    • 并非启动嵌入式 Tomcat（以 Jar 发布）
  • 添加控制器和视图代码 https://spring.io/guides/gs/serving-web-content/
    • 控制器位置 src/main/java/com.xuepub/GreetingController.java
    • 视图位置 src/main/resources/templates/greeting.html
  • 重新启动 Tomcat
  • 访问 localhost:8080/greeting?name=abc，得到结果 Hello, abc!
