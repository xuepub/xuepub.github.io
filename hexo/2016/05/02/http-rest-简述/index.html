HTTP REST 简述

HTTP 协议

目前为 HTTP 1.1 用户要访问某个资源，通过浏览器发送请求 Request，指向了 URL / URI 对应的服务器 服务器接收到请求后，返回给用户一个响应 Response 如果访问成功，通常响应会包含页面内容，如果不成功，可能会看到 404 页面   HTTP 协议是无状态的   请求类型：OPTIONS、GET、HEAD、POST、PUT、DELETE、TRACE、CONNECT 响应状态码：1xx（消息）、2xx（成功）、3xx（重定向）、4xx（客户端错误）、5xx（服务器错误）  

REST

REST 表述性状态转移，是当前最流行的互联网架构之一，目的是设计“完美 URL”，降低客户端和服务端的复杂度 通过设计 URL 来统一资源的表述，规定请求类型：

  • POST /uri 增加资源
  • GET /uri/{id} 查找资源
  • PUT /uri/{id} 更新资源
  • DELETE /uri/{id}  删除资源
  • （可选）PATCH

资源既可以是 HTML（提供内容），也可以是 JSON 数据（提供 APIs、Web Services） 可构造不同层次的 Web 系统
