AngularJS 实践心得

0. 序

2016 年下半年，我主持的 web 开发项目中，深度使用了基于 Jhipster 环境的 Angular + Spring Boot 架构， 对于 Angular 单页应用的写法有了一些心得，分享于此：  

1. 代码规范与文件结构

Jhipster 生成的前端代码，最初采用了John Papa 著名的 Angular 编码规范。 后续开发中建议延续该风格，其要点是：

  • 职责单一原则，一个 js 文件只做一件事（state、controller、service、directive）
  • 使用概念更清晰的 vm （一些连接视图-模型的对象）替换 scope， 除了在最常见的 controller 和 html 组合中使用，还应在 directive 中使用 vm
  • LIFT 原则：快速 L 定位代码、优雅 I 标识代码 、保持最 F 平等的结构、T 尝试不重造轮子

  根据项目的情况，文件结构采用了主题驱动，即将同一主题的页面文件放在同一目录下。 另设置了 app/service/ 目录，集中实现了对后端 REST 接口的 $resource 调用  

2. SEO

AngularJS 最让人头疼的莫过于 SEO。 Baiduspider 无法读取单页应用中的内容。 陆续看了知乎专栏、去哪儿网以及 angular 中文社区的设计，参考了 Prerender 的源码后，解决方案如下：

Step 1

需要先将页面中的 # 去掉，前端在 Angular 中使用 html5Mode， 后端在 Java 中写 filter（ nginx 中，可以通过修改 rewrite 实现）

Step 2

在后端的 filter 中监听首页或特定页面的 http 请求， 当发现 User-Agent 为爬虫时，通过非 Angular SPA 的方式（如 JSP - Thymeleaf），后端渲染后返回  

3. ng-show、ng-if、ng-class、ng-repeat

ng-if，用于初始化时确定显示/隐藏的内容，适用于不同 directive 的切换 ng-show，可用于各种状态下内容的显示/隐藏，但是可能有闪烁的问题 ng-class 的 visibility 可以解决某两个元素同步显示/隐藏的问题，避免了同步闪烁   闪烁的解决方案：

  • 设置 vm.dataIsReady 字段，用于数据加载完成之后，显示出闪烁部分的内容
  • 数据异步加载，一定要在回调中设置 vm.dataIsReady = true
  • 数据循环处理，需要在代码段中加上 $timeout(function( vm.dataIsReady = true; ));

  ng-repeat 在嵌套时，外层 ng-repeat 的$index 被内层覆盖，需要根据情况手动加入数据索引  

4. REST / $resource

  • 尽可能减少 REST 的次数，前后端要做出相应的调整，比如打包多个对象后传输一次
  • 禁止在循环中使用 REST，可对返回的数组进行循环
  • 多次 REST  要严格使用回调，上一次 REST 成功回调后，再执行下一次 REST

  使用 $resource 所构造的服务时，需要注意参数的设置：

  • 参数1：url 中多个参数的值对象
  • 参数2： POST / PUT 的上传对象
  • 参数3：成功回调函数
  • 参数4：失败回调函数

 

5. ui-router / directive

  • 使用 ui-router 对状态进行管理，支持 url 切换；页面局部不切换 url，建议使用 directive
  • directive 的对象双绑 “=”，函数单绑 “&”
  • 使用 $watch 对传入 directive  的对象进行监控

 

6. 其他

  • $sce.trustAsHtml() 与 ng-bind-html 使用会引起安全问题，需要有另外的 js 注入检测
  • ng 可与 d3.js 完美结合
  • 有用工具 ng-inspector
