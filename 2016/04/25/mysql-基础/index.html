一个简单实例

容器（数据库和表） 创建并使用一个书架 bookshelf 数据库：

CREATE DATABASE bookshelf;
USE bookshelf;

创建一个数据库用户 reader，对 bookshelf 进行访问：

GRANT ALL ON bookshelf.* TO ‘reader‘@’localhost’
INDETIFIED BY ‘passwd’;

在当前使用的数据库中创建表 books，并检查该表：

CREATE TABLE books (
isbn CHAR(13),
author VARCHAR(128),
title VARCHAR(128),
category VARCHAR(32),
year SMALLINT) ENGINE MyISAM;
DESCRIBE books;

删除表，检查当前库中的所有表：

DROP TABLE books;
SHOW tables;

重命名表为 works 作品、添加新列 pages 页码、删除页码列：

ALTER TABLE books RENAME works;
ALTER TABLE books ADD pages SMALLINT UNSIGNED;
ALTER TABLE books DROP pages;

  数据 数据类型支持 CHAR、VARCHAR、BINARY、TEXT、BLOB、数值型、DATE 和 TIME、AUTO_INCREMENT 自增。 为 books 表添加自增列 id，注意如果表中已经设置了 isbn 为主键，则无法添加该列，自增列必为主键：

ALTER TABLE books
ADD id INT UNSIGNED NOT NULL AUTO_INCREMENT KEY;

  增加数据 INSERT 加入一本新书：

INSERT INTO bookshelf.books (isbn, author, title, category, year)
VALUES (‘9787121022982’, ‘史蒂夫·迈克康奈尔 ‘, ‘代码大全（第2版）’, ‘编程’, ‘2006’);

  删除数据 DELETE 删除某本书：

DELETE FROM books WHERE isbn = ‘9787121022982’;

  （改）更新数据 UPDATE 更改某书的部分信息，WHRER 返回条件成立的结果用于缩小范围：

UPDATE books SET author = ‘Steve McConnell’, category = ‘code’
WHERE isbn = ‘9787121022982’;

  查询数据 SELECT 查询并显示全部图书信息：

SELECT * FROM books;

查询图书数量：

SELECT COUNT(*) FROM books;

对作者进行查询显示，重复数据只显示一次：

SELECT DISTINCT author FROM books;

查询书名中包含指定文本 “and”，并显示作者和书名信息：

SELECT author,title FROM books
WHERE title LIKE “%and%”;

限制返回结果的行数：

SELECT title FROM books LIMIT 3;

对结果逆排序：

SELECT author,title FROM books ORDER BY title DESC;

关联两张表，使用 SELECT … FROM … WHERE，或 JOIN … ON，或 NATURAL JOIN 结构   索引和主键 索引能够提高数据的搜索速度，需要为待索引列建立索引（会占用额外空间），如下限制为 author 列的前20个字符进行索引：

ALTER TABLE books ADD INDEX(author(20));

主键给每行数据唯一的标示符，在 books 表中选用 isbn 为主键，声明 isbn 列为主键必须保证数据唯一：

ALTER TABLE books ADD PRIMARY KEY(isbn);

通常在创建表时，直接添加索引和主键 如果对某字段建立了全文索引，则可以通过 MATCH … AGAINST 来高级查询
