Docker 基础

on 2016年6月20日
  • 后端
  • 运维
by szzz

 

Docker 是开源容器，用于更快的交付应用，典型场景有：

  • 代码构建
  • 应用部署
  • 容器即服务

 

参考

  • 官方网站 https://www.docker.com/
  • 中文 《第一本 Docker 书》
  • 中文 《Docker 即学即用》，Oreilly
  • 《Using Docker》、《Docker Cookbook》，Oreilly

 

入门

使用

  • 使用 Docker Toolbox，安装 Docker 软件
    • https://www.docker.com/toolbox
  • 使用 Docker Engine，在容器中运行软件镜像
    • $ docker images，查看本地镜像
    • $ docker run hello-world
  • 如果本地不存在镜像，则获取 Docker Hub 镜像

 

创建

  • 创建 Docker Hub 账户和镜像仓库
    • $ docker login
  • 创建自己的镜像
    • Dockerfile
    • $ docker build -t docker-name
  • push（推）镜像到 Docker Hub，让他人使用
    • $ docker push docker-name，上传
    • $ docker pull docker-name，下载

 

网络

  • 国内源 http://get.daocloud.io/
  • $ docker-machine restart default # Restart the environment
    $ eval $(docker-machine env default) # Refresh your environment settings

 

Docker Engine

容器操作

  • $ docker ps，查看所有正在运行中的容器列表
  • $ docker ps -a，查看所有未运行的容器列表
  • $ docker rm $(docker ps -a -q)，删除所有未运行的容器
  • $ docker start -i container-id
  • $ docker stop container-id

 

vm 操作

  • $ docker-machine ls
  • $ docker-machine create –driver virtualbox my-default
  • $ docker-machine rm my-default

 

使用预制 Ubuntu 镜像

  • $ docker pull ubuntu
  • $ docker run -i -t ubuntu /bin/bash
  • ctrl-p ctrl-q，detach 容器
  • ctrl+d，exit 容器
  • $ docker attach ubuntu-container-id
  • $ docker rmi -f ubuntu，删除镜像

 

制作 Java Spring Boot 镜像

  • https://spring.io/guides/gs/spring-boot-docker/
  • 中文 http://docs.daocloud.io/java-docker/docker-java-spring-boot

 

构建并使用 MongoDB 镜像

  • https://docs.docker.com/engine/examples/mongodb/

 

Docker Compose

用于运行多容器的 Docker 应用

  • $ docker-compose  up

示例，https://docs.docker.com/compose/wordpress/

Application.java – 1 

入口 Main 方法，执行配置及服务

类注解 @SpringBootApplication 等同于 @Configuration 配置类 + @EnableAutoConfiguration + @ComponentScan 自动扫描

@Configuration 使用基于 XML 的配置

@ComponentScan 用于自动查找所有以 @Component、@Service、@Repository、@Controller 注册的 Bean

方法注解 @Autowired 指定构造方法注入的依赖

 

domain/Customer.java – 2

POJO，即领域模型

类注解 @Entity

存储数据对象，定义属性、 get / set 方法，可通过构造方法 set 属性

 

domain/CustomerRepository.java – 3

Spring Data JPA Repositories 数据访问

继承自 Repository<Customer, Long> 或 CrudRepository<Customer, Long> 接口

 

service/CustomerService.java – 4

服务类 / 业务逻辑类 – 封装功能

类注解 @Service

 

web/CustomerController.java – 5

Spring Web MVC 控制器

类注解 @Controller 或 @RestController

类 / 方法注解 @RequestMapping 指定路由

返回视图（JSP、模板）或 Json（前后分离）

 

配置

CustomerConfig.java

支持基于 Java 和 XML 的配置

类注解 @Configuration 声明配置类，用于定义 Bean

类注解 @ComponentScan 自动扫描

方法注解 @Bean 声明配置类中方法的返回值为 Bean

路由

路由文件位置 app/Http/routes.php

<?php

Route::get('/', function () {
    return view('welcome');
});

在浏览器请求根目录（http://localhost:8000）时，通过闭包来返回视图页 resources/views/welcome.blade.php

也可以返回包含在目录中的视图页

    //指向 resource/views/about/company.blade.php
    return view('about.company');

路由传递了数据到视图

Route::get('/', function () {
    $phones = ['Apple', 'huawei', 'xiaomi'];
    return view('welcome', compact('phones'));
});

视图显示接受到的数据

<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8">
        <title>Laravel</title>
    </head>
    <body>

            @endforeach
        </ul>

路由动作 Route::get 替换为 Route::post、Route::put、Route::delete，支持多种 HTTP 请求方式

控制器

路由一般不处理逻辑和数据，而是指向控制器类，在控制器的函数中实现

php artisan make:controller PagesController

Migrations

使用 Laravel 的迁移 Migrations 功能，创建数据库表

以 books 为例，表定义见 http://www.xuepub.com/189.html，通过命令行生成迁移代码

php artisan make:migration create_books_table --create=books

位置在 database\migrations\2016_05_15_000634_create_books_table（文件中带有时间戳）

<?php

use Illuminate\Database\Schema\Blueprint;
use Illuminate\Database\Migrations\Migration;

class CreateBooksTable extends Migration
{
    public function up()
    {
        Schema::create('books', function (Blueprint $table) {
            $table->char('isbn', 13);          //isbn CHAR(13),
            $table->string('author', 128);     //author VARCHAR(128),
            $table->string('title', 128);      //title VARCHAR(128),
            $table->string('category', 128);   //category VARCHAR(32),
            $table->smallInteger('year');      //year SMALLINT
            $table->primary('isbn');           //设置主键
            $table->timestamps();              //时间戳
        });
    }

    public function down()
    {
        Schema::drop('books');
    }
}

执行 migrate命令，初始化数据库

php artisan migrate

如果数据库结构发生变化，可通过 migrate:rollback、migrate:refresh 等命令执行回滚操作

图书操作的路由设置

Route::get('books', 'BooksController@index');             //查看全部-列表 showAll
Route::get('books/create', 'BooksController@create');     //添加新书-表单
Route::post('books', 'BooksController@store');            //添加新书-命令
Route::get('books/{id}', 'BooksController@show');       //查看某书 showOne
Route::get('books/{id}/edit', 'BooksController@edit');  //修改某书-表单
Route::patch('books/{id}', 'BooksController@update');   //修改某书-命令
Route::delete('books/{id}', 'BooksController@destroy');        //删除某书-命令

表单添加图书流程：

  • 添加新路由 Route::post(‘books’, ‘BooksController@create’);
  • 控制器类 BooksController中实现 create() 方法
    • 方法至少传入一个参数 Request $request
    • new 实例化一个待存储对象
    • 用 $request 获取表单提交的数据，存入对象
    • 使用 Eloquent 中的 save() 方法存储数据
    • 返回值使用 redirect() 跳转到某个页面，或使用 back() 跳转到之前的页面
  • 模型类中要对存储的 isbn、author 等字段设置 protected $fillable = [‘isbn’, ‘author’]; 否则安全原因无法正常存储
  • 视图中实现添加表单 <form action=”books” method=”POST”>

 

表单修改数据流程：

  • 添加新路由 Route::get(‘books/{book}/edit’, ‘BooksController@edit’); 用于进入编辑页面
  • 添加新路由 Route::patch(‘books/{book}’, ‘BooksController@update’); 用于提交编辑结果
  • 实现编辑页面的控制器 update() 方法
    • use  App\Models\Book;
    • 传入至少两个参数  Request $request, Book $book
    • $book -> update($request -> all());
    • return back();
  • 实现视图
    • 表单 <form method=”POST” action=”books/{ $book->isbn }”>
    • 表单中加入 {{ method_field(‘PATCH’) }}
