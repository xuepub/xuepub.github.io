AngularJS 实践心得

on 2017年1月2日
  • 前端
by szzz

0. 序

2016 年下半年，我主持的 web 开发项目中，深度使用了基于 Jhipster 环境的 Angular + Spring Boot 架构，

对于 Angular 单页应用的写法有了一些心得，分享于此：

 

1. 代码规范与文件结构

Jhipster 生成的前端代码，最初采用了John Papa 著名的 Angular 编码规范。

后续开发中建议延续该风格，其要点是：

  • 职责单一原则，一个 js 文件只做一件事（state、controller、service、directive）
  • 使用概念更清晰的 vm （一些连接视图-模型的对象）替换 scope，
    除了在最常见的 controller 和 html 组合中使用，还应在 directive 中使用 vm
  • LIFT 原则：快速 L 定位代码、优雅 I 标识代码 、保持最 F 平等的结构、T 尝试不重造轮子

 

根据项目的情况，文件结构采用了主题驱动，即将同一主题的页面文件放在同一目录下。

另设置了 app/service/ 目录，集中实现了对后端 REST 接口的 $resource 调用

 

2. SEO

AngularJS 最让人头疼的莫过于 SEO。

Baiduspider 无法读取单页应用中的内容。

陆续看了知乎专栏、去哪儿网以及 angular 中文社区的设计，参考了 Prerender 的源码后，解决方案如下：

Step 1

需要先将页面中的 # 去掉，前端在 Angular 中使用 html5Mode，

后端在 Java 中写 filter（ nginx 中，可以通过修改 rewrite 实现）

Step 2

在后端的 filter 中监听首页或特定页面的 http 请求，

当发现 User-Agent 为爬虫时，通过非 Angular SPA 的方式（如 JSP – Thymeleaf），后端渲染后返回

 

3. ng-show、ng-if、ng-class、ng-repeat

ng-if，用于初始化时确定显示/隐藏的内容，适用于不同 directive 的切换

ng-show，可用于各种状态下内容的显示/隐藏，但是可能有闪烁的问题

ng-class 的 visibility 可以解决某两个元素同步显示/隐藏的问题，避免了同步闪烁

 

闪烁的解决方案：

  • 设置 vm.dataIsReady 字段，用于数据加载完成之后，显示出闪烁部分的内容
  • 数据异步加载，一定要在回调中设置 vm.dataIsReady = true
  • 数据循环处理，需要在代码段中加上 $timeout(function( vm.dataIsReady = true; ));

 

ng-repeat 在嵌套时，外层 ng-repeat 的$index 被内层覆盖，需要根据情况手动加入数据索引

 

4. REST / $resource

  • 尽可能减少 REST 的次数，前后端要做出相应的调整，比如打包多个对象后传输一次
  • 禁止在循环中使用 REST，可对返回的数组进行循环
  • 多次 REST  要严格使用回调，上一次 REST 成功回调后，再执行下一次 REST

 

使用 $resource 所构造的服务时，需要注意参数的设置：

  • 参数1：url 中多个参数的值对象
  • 参数2： POST / PUT 的上传对象
  • 参数3：成功回调函数
  • 参数4：失败回调函数

 

5. ui-router / directive

  • 使用 ui-router 对状态进行管理，支持 url 切换；页面局部不切换 url，建议使用 directive
  • directive 的对象双绑 “=”，函数单绑 “&”
  • 使用 $watch 对传入 directive  的对象进行监控

 

6. 其他

  • $sce.trustAsHtml() 与 ng-bind-html 使用会引起安全问题，需要有另外的 js 注入检测
  • ng 可与 d3.js 完美结合
  • 有用工具 ng-inspector

Docker 基础

on 2016年6月20日
  • 后端
  • 运维
by szzz

 

Docker 是开源容器，用于更快的交付应用，典型场景有：

  • 代码构建
  • 应用部署
  • 容器即服务

 

参考

  • 官方网站 https://www.docker.com/
  • 中文 《第一本 Docker 书》
  • 中文 《Docker 即学即用》，Oreilly
  • 《Using Docker》、《Docker Cookbook》，Oreilly

 

入门

使用

  • 使用 Docker Toolbox，安装 Docker 软件
    • https://www.docker.com/toolbox
  • 使用 Docker Engine，在容器中运行软件镜像
    • $ docker images，查看本地镜像
    • $ docker run hello-world
  • 如果本地不存在镜像，则获取 Docker Hub 镜像

 

创建

  • 创建 Docker Hub 账户和镜像仓库
    • $ docker login
  • 创建自己的镜像
    • Dockerfile
    • $ docker build -t docker-name
  • push（推）镜像到 Docker Hub，让他人使用
    • $ docker push docker-name，上传
    • $ docker pull docker-name，下载

 

网络

  • 国内源 http://get.daocloud.io/
  • $ docker-machine restart default # Restart the environment
    $ eval $(docker-machine env default) # Refresh your environment settings

 

Docker Engine

容器操作

  • $ docker ps，查看所有正在运行中的容器列表
  • $ docker ps -a，查看所有未运行的容器列表
  • $ docker rm $(docker ps -a -q)，删除所有未运行的容器
  • $ docker start -i container-id
  • $ docker stop container-id

 

vm 操作

  • $ docker-machine ls
  • $ docker-machine create –driver virtualbox my-default
  • $ docker-machine rm my-default

 

使用预制 Ubuntu 镜像

  • $ docker pull ubuntu
  • $ docker run -i -t ubuntu /bin/bash
  • ctrl-p ctrl-q，detach 容器
  • ctrl+d，exit 容器
  • $ docker attach ubuntu-container-id
  • $ docker rmi -f ubuntu，删除镜像

 

制作 Java Spring Boot 镜像

  • https://spring.io/guides/gs/spring-boot-docker/
  • 中文 http://docs.daocloud.io/java-docker/docker-java-spring-boot

 

构建并使用 MongoDB 镜像

  • https://docs.docker.com/engine/examples/mongodb/

 

Docker Compose

用于运行多容器的 Docker 应用

  • $ docker-compose  up

示例，https://docs.docker.com/compose/wordpress/

on 2016年6月15日

AngularJS 1 即将退役，由基于 TypeScript 的 Angular 2 取代

旨在开发 SPA 单页应用

  • 官方文档 https://docs.angularjs.org/guide
  • 入门 http://w3schools.bootcss.com/angular/default.html
  • 中文参考 https://github.com/jmcunningham/AngularJS-Learning/blob/master/ZH-CN.md

 

图书

  • 《AngularJS 学习手册》，Oreilly
  • 《用 AngularJS 开发下一代 Web 应用》，Oreilly
  • 《AngularJS 权威教程》，ng-book

 

核心概念

  • Template 模板：附加标记的 HTML
  • Directives 指令：自定义属性和元素的扩展 HTML
  • Model 模型：视图中显示给用户的数据，并与用户交互
  • Scope 域：存储模型的上下文，以便于控制器指令和表达式访问 $scope
  • Expressions 表达式：从域中访问变量和函数 {{ expression }}
  • Compiler 编译器：解析模板、初始化指令和表达式
  • Filter 过滤器：为用户显示格式化表达式值 {{ expression | filter }}
  • View 视图：用户所见（DOM）
  • Data Binding 数据绑定：在模型和视图之间同步数据
  • Controller 控制器：视图后的商业逻辑
  • Dependency Injection 依赖注入：创建并连接对象和函数
  • Injector 注入器：依赖注入容器
  • Module 模块：一个容器用于应用的不同部分，包括控制器、服务、过滤器、指令，由注入器配置
  • Service 服务：依赖视图的可重用商业逻辑

 

常用指令

  • ng-app 声明 AngularJS 应用的根元素，其中指令可用于声明绑定和定义行为
  • ng-bind 设置 DOM 元素的文本为表达式值。如 <span ng-bind=”name”></span> 在 span 元素中显示了 name 的值，任何在应用域中变量 name 的更改都立刻反应到 DOM
  • ng-model 与 ng-bind 相似，但在视图和域之间建立了一个双向数据绑定
  • ng-model-options 提供了如何做模型更新的调整
  • ng-class 让 class 属性动态加载
  • ng-controller 指定一个 JavaScript 控制器类，对 HTML 表达式求值
  • ng-repeat 从一个集合中每项实例化一个元素
  • ng-show & ng-hide 有条件的显示或隐藏元素，根据布尔表达式的值。显示和隐藏通过设置 CSS display 样式来完成
  • ng-switch 有条件的从一系列选择中实例化一个模板，根据选择表达式的值
  • ng-view 这一基本指令负责处理路由，由特定的控制器驱动，在渲染模板之前解析 JSON
  • ng-if 基本的 if 表达式指令，允许显示是否条件为 true 的元素。当条件为false，元素将从 DOM 移除。当 true 重新插入编译过的元素备份
  • ng-aria 一个模块用来支持通用 ARIA 属性
  • ng-animate 一个模块提供了 JavaScript、CSS3 过渡和 CSS3 关键帧动画的支持，连接已有的核心和自定义指令

引入

<script src="https://cdn.bootcss.com/react/15.1.0/react.js"></script>

Chrome 中的 React Developer Tools，用于调试 Web Server 发布的 React 页面

 

基本概念

第一个 React 程序

创建自定义组件 component

React.createClass() 用来创建新的 React 组件 <Hello>，

其中的 render 渲染函数返回 JSX 标记，

ReactDOM.render() 使用新组件 Hello，注入至 #content

<body>
    <div id="content"></div>
    <script type="text/babel">
        //这里是 React 代码
        var Hello = React.createClass({
            render: function() {
                return (
                    <div>
                        <h1>Hello</h1>
                    </div>
                );
            }
        });
        ReactDOM.render(
            <Hello />,
            document.getElementById('content')
        );
    </script>
</body>

 

 

{ }

JSX 中的 { } 能够嵌入 JS 对象，即动态信息

获取 JS 变量，包括属性 {this.props.foo}、状态 {this.state.foo}、子节点 {this.props.children}

获取 JS 函数的返回值 {this.f()}，或触发事件

var s = 'Sunny';
var Hello = React.createClass({
    render: function() {
        return (
            <div>          
                <h1>Hello {this.props.name}!</h1>
                <h2>{this.props.children}, {s}</h2>  
            </div> 
        );
    }
});
ReactDOM.render(
    <Hello name="szzz">Children node content.</Hello>,
    document.getElementById('content')
);<br>

 

加载 JSON 数据

通过函数式编程的 map() 函数，遍历数组

声明 key 属性为 React 所要求：每个子元素都要有，不然出现 error

var books = [
    {id: 1, name: "Code Complate 2"},
    {id: 2, name: "Thinking in Java"}
];
var Hello = React.createClass({
    render: function() {	
        return (
            <div>          
                {this.props.data.map(function(book, index) {
    		    return (
    			<li key={index}>
    			    {book.id} {book.name}
    			</li>
    		    );
    		})}  
            </div> 
        );
    }
});
ReactDOM.render(
    <Hello data={books} />,
    document.getElementById('content')
);

 

JSX

JSX 本质上是 JavaScript 的语法转换，可创建语义丰富的标签

//JSX
var app = <App name="Mike">Hello</App>;
//JS
var app = React.createElement(App, {name:"Mike"}, 'Hello');

一些关注点：

  • JSX 用小写字母开头的标签名渲染 HTML 标签（字符串），大写字母开头的标签名渲染 React 组件（类）
  • { } 中使用 condition ? true-branch : false-branch 三目运算符 ，实现 if-else 结构
  • { } 中使用 && 运算符，当前操作数为 true 输出后操作数
  • 事件 onChange、onClick
  • 特殊属性
    • key，用于调整组件位置
    • ref，引用子组件
    • dangerouslySetInnerHTML
    • htmlFor
    • className

 

组件生命周期

  • 生命周期：实例化、活动期、销毁
  • 实例化 / 加载：getDefaultProps、getInitialState、componentWillMount、render、componentMidMount
  • 活动期 / 更新：componentWillReceiveProps、shouldComponentUpdate、componentWillMount、render、componentMidMount
  • 销毁 / 移除：componentWillUnmount

props 在 getDefaultProps 初始化时传入数据，此后只读，即单向数据流

state 中存储了视图状态，在组件事件中修改 this.setState() 是改变视图的关键点

 

复杂组件

  • 复杂组件可由多个简单组件组合而成，这些组件形成了树结构
  • 大型项目一般从下而上（也可以理解为从内到外、从小到大）实现，便于写测试
  • 组件的初始数据通过 props 传入，从上到下单向数据流动，
  • state 则存储了运行组件时变化的状态值，包括可能被组件的事件处理器改变并触发用户界面更新的数据，如搜索框内输入的文本、当前多选框的选中值
  • 让尽可能多的组件无状态化

Application.java – 1 

入口 Main 方法，执行配置及服务

类注解 @SpringBootApplication 等同于 @Configuration 配置类 + @EnableAutoConfiguration + @ComponentScan 自动扫描

@Configuration 使用基于 XML 的配置

@ComponentScan 用于自动查找所有以 @Component、@Service、@Repository、@Controller 注册的 Bean

方法注解 @Autowired 指定构造方法注入的依赖

 

domain/Customer.java – 2

POJO，即领域模型

类注解 @Entity

存储数据对象，定义属性、 get / set 方法，可通过构造方法 set 属性

 

domain/CustomerRepository.java – 3

Spring Data JPA Repositories 数据访问

继承自 Repository<Customer, Long> 或 CrudRepository<Customer, Long> 接口

 

service/CustomerService.java – 4

服务类 / 业务逻辑类 – 封装功能

类注解 @Service

 

web/CustomerController.java – 5

Spring Web MVC 控制器

类注解 @Controller 或 @RestController

类 / 方法注解 @RequestMapping 指定路由

返回视图（JSP、模板）或 Json（前后分离）
