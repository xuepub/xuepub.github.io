环境配置

  • Windows 10
  • Sublime Text / PhpStorm

composer global require "laravel/installer"

（出现无法连接时使用 composer config secure-http false）
（composer config -g github-oauth.github.com [tokens]）

启动

建立新工程，目前为 5.2

laravel new projectname

配置 storage 和 bootstrap/cache 访问权限为可写

.env 中设置 key

php artisan key:generate

配置 config/app.php

启动内置 Web Server （端口 8000）来开发调试页面

php artisan serve

 

路由

路由文件位置 app/Http/routes.php

<?php

Route::get('/', function () {
    return view('welcome');
});

在浏览器请求根目录（http://localhost:8000）时，通过闭包来返回视图页 resources/views/welcome.blade.php

也可以返回包含在目录中的视图页

    //指向 resource/views/about/company.blade.php
    return view('about.company');

路由传递了数据到视图

Route::get('/', function () {
    $phones = ['Apple', 'huawei', 'xiaomi'];
    return view('welcome', compact('phones'));
});

视图显示接受到的数据

<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8">
        <title>Laravel</title>
    </head>
    <body> 
        <ul>
            @foreach ($phones as $phone)
                <li>{{ $phone }}</li>
            @endforeach
        </ul>
    <body>
</body>

路由动作 Route::get 替换为 Route::post、Route::put、Route::delete，支持多种 HTTP 请求方式

路由参数，通过 URI 获取参数

Route::get('user/{id}', function ($id) {
    return 'User '.$id;
});

 

控制器

路由一般不处理逻辑和数据，而是指向控制器类，在控制器的函数中实现

<?php

Route::get('/', 'PagesController@home');

PagesController@home  指向 app/Http/Controllers/PagesController.php 的 PagesController->home() 函数

可使用命令行创建控制器

php artisan make:controller PagesController
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use App\Http\Requests;

class PagesController extends Controller
{
    public function home
    {
        $phones = array('Apple', 'huawei', 'xiaomi'); 
        return view('welcome', compact('phones'));
    }
}

 

视图

视图采用 Blade 引擎，通过 @yield 实现模板继承

在布局页面（模板页面）中使用 @yield，以显示继承页面的 @section 内容

<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8">
        <title>Laravel</title>
    </head>
    <body>      
        @yield('content')       
    </body>
</html>

@extends('layout')

@section('content')
    <ul>
        @foreach ($phones as $phone)
            <li>{{ $phone }}</li>
        @endforeach
    </ul>
@stop

包括布局、侧边栏等抽取出来的部分视图代码，可重复使用

Migrations

使用 Laravel 的迁移 Migrations 功能，创建数据库表

以 books 为例，表定义见 http://www.xuepub.com/189.html，通过命令行生成迁移代码

php artisan make:migration create_books_table --create=books

位置在 database\migrations\2016_05_15_000634_create_books_table（文件中带有时间戳）

<?php

use Illuminate\Database\Schema\Blueprint;
use Illuminate\Database\Migrations\Migration;

class CreateBooksTable extends Migration
{
    public function up()
    {
        Schema::create('books', function (Blueprint $table) {
            $table->char('isbn', 13);          //isbn CHAR(13),
            $table->string('author', 128);     //author VARCHAR(128),
            $table->string('title', 128);      //title VARCHAR(128),
            $table->string('category', 128);   //category VARCHAR(32),
            $table->smallInteger('year');      //year SMALLINT
            $table->primary('isbn');           //设置主键
            $table->timestamps();              //时间戳
        });
    }

    public function down()
    {
        Schema::drop('books');
    }
}

执行 migrate命令，初始化数据库

php artisan migrate

如果数据库结构发生变化，可通过 migrate:rollback、migrate:refresh 等命令执行回滚操作

Eloquent ORM

使用 Eloquent ORM 模型，先创建 Book 模型类

php artisan make:model Book

生成 app/Book.php，内容可以不进行任何修改，将生成文件移动至 app/Models/Book.php

<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Book extends Model
{
    //
}

显示数据的开发流程：Route – Controller – View

Route::get('books', 'BooksController@index');
<?php

namespace App\Http\Controllers;

use App\Models\Book; //Eloquent 引入
use Illuminate\Http\Request;
use App\Http\Requests;

class BooksController extends Controller
{
    public function index()
    {
    	$books = Book::All(); //Eloquent 
    	return view('books.index', compact('books'));
    }
}
//View 模板在 resources/Views/layout.blade.php
@extends('layout')

@section('content')
	@foreach ($books as $book)
		<h1>{{ $book->title }}</h1>
	@endforeach
@stop

图书操作的路由设置

Route::get('books', 'BooksController@index');             //查看全部-列表 showAll

Route::delete('books/{id}', 'BooksController@destroy');        //删除某书-命令

表单添加图书流程：

  • 添加新路由 Route::post(‘books’, ‘BooksController@create’);
  • 控制器类 BooksController中实现 create() 方法
    • 方法至少传入一个参数 Request $request
    • new 实例化一个待存储对象
    • 用 $request 获取表单提交的数据，存入对象
    • 使用 Eloquent 中的 save() 方法存储数据
    • 返回值使用 redirect() 跳转到某个页面，或使用 back() 跳转到之前的页面
  • 模型类中要对存储的 isbn、author 等字段设置 protected $fillable = [‘isbn’, ‘author’]; 否则安全原因无法正常存储
  • 视图中实现添加表单 <form action=”books” method=”POST”>

 

表单修改数据流程：

  • 添加新路由 Route::get(‘books/{book}/edit’, ‘BooksController@edit’); 用于进入编辑页面
  • 添加新路由 Route::patch(‘books/{book}’, ‘BooksController@update’); 用于提交编辑结果
  • 实现编辑页面的控制器 update() 方法
    • use  App\Models\Book;
    • 传入至少两个参数  Request $request, Book $book
    • $book -> update($request -> all());
    • return back();
  • 实现视图
    • 表单 <form method=”POST” action=”books/{ $book->isbn }”>
    • 表单中加入 {{ method_field(‘PATCH’) }}
