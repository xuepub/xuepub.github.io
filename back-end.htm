Docker 基础

on 2016/06/20
  • 后端
  • 运维
by szzz

 

Docker 是开源容器，用于更快的交付应用，典型场景有：

  • 代码构建
  • 应用部署
  • 容器即服务

 

参考

  • 官方网站 https://www.docker.com/
  • 中文 《第一本 Docker 书》
  • 中文 《Docker 即学即用》，Oreilly
  • 《Using Docker》、《Docker Cookbook》，Oreilly

 

入门

使用

  • 使用 Docker Toolbox，安装 Docker 软件
    • https://www.docker.com/toolbox
  • 使用 Docker Engine，在容器中运行软件镜像
    • $ docker images，查看本地镜像
    • $ docker run hello-world
  • 如果本地不存在镜像，则获取 Docker Hub 镜像

 

创建

  • 创建 Docker Hub 账户和镜像仓库
    • $ docker login
  • 创建自己的镜像
    • Dockerfile
    • $ docker build -t docker-name
  • push（推）镜像到 Docker Hub，让他人使用
    • $ docker push docker-name，上传
    • $ docker pull docker-name，下载

 

网络

  • 国内源 http://get.daocloud.io/
  • $ docker-machine restart default # Restart the environment
    $ eval $(docker-machine env default) # Refresh your environment settings

 

Docker Engine

容器操作

  • $ docker ps，查看所有正在运行中的容器列表
  • $ docker ps -a，查看所有未运行的容器列表
  • $ docker rm $(docker ps -a -q)，删除所有未运行的容器
  • $ docker start -i container-id
  • $ docker stop container-id

 

vm 操作

  • $ docker-machine ls
  • $ docker-machine create –driver virtualbox my-default
  • $ docker-machine rm my-default

 

使用预制 Ubuntu 镜像

  • $ docker pull ubuntu
  • $ docker run -i -t ubuntu /bin/bash
  • ctrl-p ctrl-q，detach 容器
  • ctrl+d，exit 容器
  • $ docker attach ubuntu-container-id
  • $ docker rmi -f ubuntu，删除镜像

 

制作 Java Spring Boot 镜像

  • https://spring.io/guides/gs/spring-boot-docker/
  • 中文 http://docs.daocloud.io/java-docker/docker-java-spring-boot

 

构建并使用 MongoDB 镜像

  • https://docs.docker.com/engine/examples/mongodb/

 

Docker Compose

用于运行多容器的 Docker 应用

  • $ docker-compose  up

示例，https://docs.docker.com/compose/wordpress/

Application.java – 1 

入口 Main 方法，执行配置及服务

类注解 @SpringBootApplication 等同于 @Configuration 配置类 + @EnableAutoConfiguration + @ComponentScan 自动扫描

@Configuration 使用基于 XML 的配置

@ComponentScan 用于自动查找所有以 @Component、@Service、@Repository、@Controller 注册的 Bean

方法注解 @Autowired 指定构造方法注入的依赖

 

domain/Customer.java – 2

POJO，即领域模型

类注解 @Entity

存储数据对象，定义属性、 get / set 方法，可通过构造方法 set 属性

 

domain/CustomerRepository.java – 3

Spring Data JPA Repositories 数据访问

继承自 Repository<Customer, Long> 或 CrudRepository<Customer, Long> 接口

 

service/CustomerService.java – 4

服务类 / 业务逻辑类 – 封装功能

类注解 @Service

 

web/CustomerController.java – 5

Spring Web MVC 控制器

类注解 @Controller 或 @RestController

类 / 方法注解 @RequestMapping 指定路由

返回视图（JSP、模板）或 Json（前后分离）

HTTP REST 简述

on 2016/05/02
  • 后端
  • 开发
by szzz

HTTP 协议

目前为 HTTP 1.1

用户要访问某个资源，通过浏览器发送请求 Request，指向了 URL / URI 对应的服务器

服务器接收到请求后，返回给用户一个响应 Response

如果访问成功，通常响应会包含页面内容，如果不成功，可能会看到 404 页面

 

HTTP 协议是无状态的

 

请求类型：OPTIONS、GET、HEAD、POST、PUT、DELETE、TRACE、CONNECT

响应状态码：1xx（消息）、2xx（成功）、3xx（重定向）、4xx（客户端错误）、5xx（服务器错误）

 

REST

REST 表述性状态转移，是当前最流行的互联网架构之一，目的是设计“完美 URL”，降低客户端和服务端的复杂度

通过设计 URL 来统一资源的表述，规定请求类型：

  • POST /uri 增加资源
  • GET /uri/{id} 查找资源
  • PUT /uri/{id} 更新资源
  • DELETE /uri/{id}  删除资源
  • （可选）PATCH

资源既可以是 HTML（提供内容），也可以是 JSON 数据（提供 APIs、Web Services）

可构造不同层次的 Web 系统

环境

OS : Windows 10 1511

Python -v : 3.5.1 win32

IDE : Pychram 2016.1.2

Test Browser : Firefox

DB : MySQL

 

模块准备

urllib 内置，获取 HTML，容易被封

Selenium 需安装，配合浏览器获取 HTML

BeatifulSoup4 需安装，文档，分析 HTML

pymysql 需安装，操作 MySQL

 

思路

优先使用 urllib 进行网页抓取，如果完全被禁，使用 Selenium，后者默认使用带 UI 的浏览器 Firefox ，可使用无 UI 浏览器取代。

Selenium / urllib 取出数据所在的 HTML 代码，交给 BeatifulSoup4 做具体抽取

可通过列表（数组）、字典（关联数组），或类来存储页面上的结构化数据

将内存中的数据通过 pymysql 存入 MySQL 数据库

 

一些细节

Selenium

设置加载页面的时间，防止被墙的脚本拖延页面加载时间  driver.set_page_load_timeout(num)

使用 XPath 获取元素 driver.find_element_by_xpath(‘//body’)

得到元素 HTML 代码 body.get_attribute(‘innerHTML’)

BeatifulSoup4

使用 urllib.parse.unquote() 对链接中的 URL 转码进行还原

SQL

使用 INSERT IGNORE INTO 取代 INSERT INTO，避免主键重复造成的错误

在 Ubuntu Apache2 的 WordPress 网站中，固定链接需要如下设置：

    • sudo service apache2 restart
  2. 启用 .htaccess
    • 修改 apache2 配置文件，sudo nano /etc/apache2/sites-enabled/000-default.conf
      其中内容为
      <Directory /var/www/html>
      Options Indexes FollowSymLinks MultiViews

      </Directory>
    • sudo service apache2 restart
